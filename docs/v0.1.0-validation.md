## Browser Validation

Browser validation was performed by committed Playwright tests through `pnpm --dir web test:e2e`.

The suite fails on uncaught page errors, unhandled request failures to required API/health endpoints, and unexpected console errors. It uses:

- a temporary SQLite database;
- the production-built frontend;
- the Go server command;
- the Go worker command;
- a mock `/v1/models` and `/v1/chat/completions` provider;
- a mock HTTP MCP server with `tools/list` and `tools/call`.

## Security Review Notes

- Provider and MCP secret values remain write-only from the frontend and are not returned by API responses.
- `APP_ENCRYPTION_KEY` is required as a valid 32-byte base64 value in production.
- Diagnostics expose counts, statuses, timestamps, and build metadata, not provider keys, MCP authorization headers, or database credentials.
- Docker runtime remains non-root, read-only, capability-dropped, and without Docker socket or privileged mode.
- MCP stdio execution remains backend-controlled without shell interpolation.
- Chat regeneration guidance is now sent as internal provider context and is not displayed as a user message.
- Task tool-call results are bounded and persisted with structured state for task-run details.
